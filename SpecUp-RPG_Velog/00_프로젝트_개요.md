# SpecUp RPG 🎮 — 프로젝트 개요

> "의지가 없어도 굴러가게 만드는 시스템"

---

## 🔥 왜 이 프로젝트를 시작했나

공부를 시작하려고 마음먹었는데 막상 뭘 해야 할지 모르겠고,
겨우 시작해도 며칠 못 가서 흐지부지된 경험, 다들 한 번씩 있지 않나요?

저도 그랬습니다.

결심하기까지 오래 걸리고, 막상 시작해도 금방 포기하고,
혼자서는 동기 유지가 너무 어려웠습니다.

기존 서비스들(Habitica, 열품타 등)을 찾아봤지만
이미 의지가 있는 사람들을 위한 도구라는 느낌이 강했습니다.

> **"의지가 있어야 쓸 수 있는 앱이 아니라, 앱이 의지를 만들어주는 서비스"**

이 한 문장이 SpecUp RPG를 기획하게 된 이유입니다.

---

## 🎯 서비스 핵심 컨셉

**포기가 빠르고 시작이 어려운 사람들을 위한 AI 맞춤형 성장 RPG 플랫폼**

게임에서 퀘스트를 깨듯, 매일 할 일을 퀘스트로 전환하고
캐릭터를 키우고 펫을 돌보면서 자연스럽게 스펙을 쌓아가는 서비스입니다.

### 기존 서비스와 다른 점

| | 기존 서비스 (Habitica 등) | SpecUp RPG |
|---|---|---|
| 퀘스트 | 직접 등록해야 함 | AI가 진단 후 자동 생성 |
| 동기부여 | 스스로 만들어야 함 | 펫, 알람, 피드백이 끌어당김 |
| 실력 진단 | 없음 | AI가 대화로 레벨 판단 |
| 피드백 | 없음 | 일별/주별/월별/연별 AI 리포트 |
| 목표 변경 | 처음부터 다시 | 병합/재조정/일시정지 지원 |

## ✨ 핵심 기능 요약

### 1. AI 레벨 진단 & 맞춤 커리큘럼
가입 후 AI와 짧은 대화로 현재 실력, 목표, 가용 시간을 파악하고
그에 맞는 퀘스트 커리큘럼을 자동 생성합니다.

- 초보 개발자 → HTML/CSS 기초 4주 커리큘럼
- 중급 개발자 → Spring JPA 심화 6주 커리큘럼
- 개발자가 아니어도 OK → 디자이너, 수험생, 어학 학습자 모두 가능

### 2. RPG 퀘스트 시스템
- 일일 퀘스트 / 주간 퀘스트 / 도전 과제
- 완료 시 XP & 골드 지급 → 레벨업

### 3. 2축 레벨 시스템
```
계정 레벨 — 퀘스트 누적 달성, 절대 리셋 없음 (성실도 반영)
직업 Tier — AI 진단으로 초기값 설정 (실력 반영, 1~5단계)
```
중급 개발자가 가입하면 계정 Lv.1이지만 직업 Tier 3에서 시작합니다.

### 4. 목표 유연성
- 목표 완전 전환 → AI 재진단 후 새 커리큘럼 생성
- 단기 목표 추가 → 기존 커리큘럼 일시정지 후 병렬 진행
- 비슷한 목표 병합 → AI가 감지해서 하나로 재조정
- 페이스 조정 → 말 한마디로 퀘스트 양 조절

### 5. 캐릭터 & 펫
- 장비, 코스튬, 칭호로 캐릭터 커스텀
- 퀘스트 완료 시 펫 성장 → 7일 연속 달성 시 진화
- 3일 방치 시 펫 컨디션 저하 → 복귀 유도
- AI 펫 대화 ("오늘 커밋 아직 안 했죠? 저 배고파요 ㅠ")

### 6. AI 피드백 & 성장 예측
| 주기 | 내용 |
|---|---|
| 매일 밤 | 오늘 달성률, XP 획득, 펫 컨디션 요약 |
| 매주 일요일 | 주간 달성률 분석, 다음 주 커리큘럼 조정 제안 |
| 매월 말 | 성장 영역 분석 + 현실적 위치 진단 |
| 매년 1월 | 1년 성장 그래프 + 공유 가능한 연간 회고 카드 |

> "현재 페이스면 6개월 후 Spring Boot 중급으로 취업 지원 가능해요."

### 7. 스마트 알람
강요가 아닌 끌어당기는 알람 설계
- "뭉이가 3일째 밥을 못 먹었어요. 5분만 시간 내줄 수 있어요?"
- "XP 50만 더 모으면 Lv.15 달성이에요. 퀘스트 하나만 더!"

## 🏗️ 기술 스택

### Backend
- **Java 17** / **Spring Boot 3.x**
- **Spring Security** + **JWT** (인증/인가)
- **Spring Data JPA**
- **Spring Scheduler** (일일 퀘스트 초기화, 알람 발송)
- **MariaDB**

### Frontend
- **Next.js** / **React** / **TypeScript**
- **Tailwind CSS**

### AI
- **Gemini API** (무료 티어로 시작 → 추후 Claude API 전환 가능)
- AiService 인터페이스로 추상화 → AI 제공사 교체 시 구현체만 변경

### 배포 (예정)
- **Docker**
- **Oracle Cloud Free Tier** (서버 무료)

## 🗄️ 데이터베이스 설계 요약

총 15개 테이블로 구성됩니다.

| 영역 | 테이블 |
|---|---|
| 회원 | users, user_status, user_job_mastery |
| 퀘스트 | quest, user_quest, quest_completion |
| AI | ai_diagnosis, ai_curriculum, ai_feedback |
| 캐릭터 | pet, item, user_item |
| 칭호 | achievement, user_achievement |
| 알람 | alarm_setting |

**핵심 설계 포인트**
- 퀘스트 중복 완료 방지: `UNIQUE KEY (user_quest_id, completed_date)` + 서비스 레이어 이중 방어
- AI 제공사 교체 가능: `AiService` 인터페이스로 추상화
- 카테고리 확장: `category` 컬럼 하나로 개발자 → 디자이너 → 수험생 등 자유롭게 확장

## 🚀 개발 전략

### 단계별 개발 (기능별 백엔드 → 프론트엔드)
```
1단계: 로그인 / 회원가입
2단계: 퀘스트 시스템 (핵심 MVP)
3단계: AI 진단 & 커리큘럼
4단계: 캐릭터 & 펫
5단계: 피드백 & 알람
6단계: 소셜 (길드, 챌린지, 랭킹)
7단계: 모바일 앱 (React Native 또는 Capacitor)
```

한 기능씩 백엔드 → 프론트엔드 순서로 완성한 뒤 다음 기능으로 넘어갑니다.
눈에 보이는 결과를 빠르게 확인하며 진행하는 방식입니다.

### AI 비용 전략
- 개발 초기: Gemini API 무료 티어 사용
- AI 토큰 시스템 자체 구현 (24시간마다 자동 충전)
- 월 API 한도 캡 설정으로 비용 원천 차단
- AiService 인터페이스로 추상화 → 나중에 Claude API로 교체 용이
